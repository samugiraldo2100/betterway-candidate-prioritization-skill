Submission summary:
This repository contains a Claude Skill designed to prioritize candidates against two job descriptions. The skill supports a recruiter-facing workflow that validates input files, interprets both job descriptions, evaluates every candidate against Job A and Job B, applies a weighted rubric, assigns a final classification, prioritizes recruiter outreach, and prepares a structured final report.

The project was designed with traceability, privacy, and responsible screening in mind. Candidate documents, job PDFs, extracted text, generated inventories, workspace outputs, and generated reports are intentionally kept local and ignored by Git.

Main deliverable files:
- candidate-prioritization/SKILL.md
- candidate-prioritization/references/
- candidate-prioritization/scripts/
- reports/templates/candidate-prioritization-report.md
- agent.md
- docs/notion-mcp-setup.md
- SUBMISSION.txt

What the skill does:
- Validates that the input package contains the required job and candidate files.
- Separates the evaluation process into clear workflow phases.
- Evaluates each candidate against both Job A and Job B.
- Uses evidence states to avoid unsupported assumptions.
- Applies a weighted candidate evaluation rubric.
- Classifies candidates as Job A, Job B, Both, or Neither.
- Assigns recruiter outreach priority.
- Produces a structured recruiter-facing report template.
- Includes validation scripts to verify project structure and reference integrity.
- Supports optional publication to Notion when a Notion MCP connection is available.

What the skill does not do:
- It does not make final hiring decisions.
- It does not replace human recruiter review.
- It does not rank candidates using protected or sensitive personal characteristics.
- It does not commit private candidate files or generated outputs to Git.
- It does not publish to Notion unless a valid Notion MCP connection is configured.
